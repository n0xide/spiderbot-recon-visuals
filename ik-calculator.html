The math (concepts.md §2)

// 3-DOF closed-form IK — runs in ~1 µs per call on Teensy 4.1
// Foot target (x, y, z) in body frame, all in mm.
// Femur = 110, tibia = 130, coxa_len = 30 (offset from coxa pivot to femur pivot)

theta_coxa = atan2(y, x);

// Project target into the leg's vertical plane
r = sqrt(x*x + y*y) - coxa_len;   // horizontal distance from femur joint
h = z;                            // vertical distance from femur joint
L = sqrt(r*r + h*h);              // straight-line distance to foot

if (L > femur + tibia)            // unreachable — clamp or fail
  return WORKSPACE_VIOLATION;

// Law of cosines on the femur-tibia triangle
alpha = acos((femur*femur + L*L - tibia*tibia) / (2.0 * femur * L));
beta  = atan2(h, r);
theta_femur = alpha + beta;

gamma = acos((femur*femur + tibia*tibia - L*L) / (2.0 * femur * tibia));
theta_tibia = M_PI - gamma;

// Apply per-servo offsets and direction before sending to bus
servo_cmd[coxa]  = (theta_coxa  + offset_coxa)  * dir_coxa;
servo_cmd[femur] = (theta_femur + offset_femur) * dir_femur;
servo_cmd[tibia] = (theta_tibia + offset_tibia) * dir_tibia;

Body-pose superposition

The robot's 6-DOF body pose (roll, pitch, yaw, x, y, z) is superimposed onto foot targets before per-leg IK. A homogeneous transform takes the desired foot position in world frame and rotates/translates it into the body frame for each leg.

// Pseudocode
foot_world  = current_foot_position;          // where the foot is on the ground
T_world_to_body = inverse(body_pose);         // body pose homogeneous transform
foot_body  = T_world_to_body * foot_world;    // foot in body frame
joint_cmd  = ik(foot_body);                   // closed-form 3-DOF IK above

This is what makes the body-twist gait possible: feet stay planted in world frame, body translates/rotates, each leg's IK absorbs the relative motion.
