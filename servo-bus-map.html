Addressing procedure (do this once, before chassis assembly)

  1. Plug single LX-224HV into the Hiwonder programmer cable.
  2. Open the Hiwonder Bus Servo Debug Tool (or use your firmware utility) on the dev laptop.
  3. Read current address (factory default 0x01).
  4. Set new address per the table above. Verify by re-reading.
  5. Apply a vinyl label to the servo case with the address (Sharpie ink wears off; vinyl doesn't).
  6. Set servo to the safe-pose angle (e.g., 150° / center) and lock to that pose during chassis assembly so it doesn't try to fight you.
  7. Repeat for all 24. Track which are addressed in log.md.

⚠ Hot-swap field spares should be pre-addressed for the most likely failure positions (front legs see the most abuse). Or carry the programmer cable in the field kit.

Each sub-bus electrical

Per sub-bus components

  • 22 µF tantalum + ferrite bead on power tap (12 V from Buck #1)
  • TVS SMAJ24CA across power for kickback protection
  • 3-wire JST daisy-chain to 6 servos (3 per leg × 2 legs)
  • Optional 120 Ω termination at far end if scope shows ringing
  • Bus drive: Teensy TX1 through 74HC126 enable buffer or direct half-duplex from UART1 — decide at P0 bench

Failure isolation

A short on one sub-bus drops 6 servos (2 legs on one side) but leaves the other 18 alive. The robot can still walk — at reduced gait fidelity — long enough to reach an estop or operator.

Compare to a single-bus topology: one short kills all 24 servos and the robot collapses. Sub-bus partitioning is cheap insurance.

Per-servo calibration (recorded after assembly)

Each servo needs zero-offset, direction (+1 / −1 for mirrored sides), and software joint limits. Procedures in calibration checklist. Stored in config/servo_offsets.yaml + config/joint_limits.yaml.

leg_R1:                    # front-right leg
  coxa:   {address: 0x01, offset_deg: -2.3, direction: +1, min_deg: -55, max_deg:  55}
  femur:  {address: 0x02, offset_deg:  4.1, direction: +1, min_deg: -15, max_deg:  85}
  tibia:  {address: 0x03, offset_deg: -1.8, direction: +1, min_deg:   5, max_deg: 125}
leg_R2:
  coxa:   {address: 0x04, offset_deg:  0.7, direction: +1, ...}
